# Data Annotation Project – Amazon Product Reviews

## Objective
This project demonstrates a real-world data annotation workflow 
for classifying Amazon product reviews into three sentiment 
categories: positive, negative, and neutral.

The goal is to simulate how data is labeled, validated, and 
prepared for machine learning models.

All annotations were manually performed using defined guidelines 
to ensure consistency and accuracy.
This project simulates a real-world data annotation workflow 
used in machine learning data preparation pipelines.

---

## Dataset
- Source: Sample Amazon-style product reviews
- Size: 50+ reviews
- Format: CSV

---

## Annotation Process
 Ensured high accuracy and consistency while labeling data 
according to predefined guidelines
---

## Labels
- positive → satisfied / happy reviews
- negative → complaints / dissatisfaction
- neutral → mixed or unclear sentiment

---

## Quality Checks
- Checked for missing labels
- Reviewed duplicate entries
- Verified label consistency across dataset
- Re-checked ambiguous cases using annotation guidelines
- Maintained accuracy and consistency in labeling decisions
- Spot-checked a subset of labeled data to validate 
annotation accuracy
